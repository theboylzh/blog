一、暗黑模式的前世今生¶

什么是暗黑模式¶

暗黑模式是指软件的深色界面。这里需要注意，暗黑模式和护眼模式是两种概念。

  • 暗黑模式还要关注在强光下的视觉识别度
  • 护眼模式主要用于暗光环境，注重阅读舒适性

由于网上的资料众多，这里就不展开讲了。概念不是重点。

暗黑模式要注意什么¶

正如上文所说，暗黑模式的使用场景和亮色模式差不多一致。因此保证视觉识别度是很重要的事情。这里有几点需要注意：

  • 减轻眼睛压力。避免大面积使用纯黑和纯白，这样对比度过高，使用久了会刺眼
  • 注重可读性。重要的文字和元素需要满足无障碍标准中的 AA 级别；视觉层次要有区分
  • 营造高端感。这也是暗黑模式带来的一种常见的感觉

涉及知识¶

WCAG2.0 标准¶

  • 常规文字至少满足 AA 级标准，最低对比度为：4.5:1
  • 大本文的对比度至少满足 3:1
  • 图标和组件至少满足 AA 级标准，最低对比度为 3:1

亮度和对比度计算公式¶

颜色对比度计算：对比度 = (L1 + 0.05) / (L2 + 0.05)

相对亮度计算：L = 0.2126 * R’ + 0.7152 * G’ + 0.0722 * B’

相对亮度是这样计算的：

  1. 计算 sRGB。对于每个 RGB（红绿蓝）值（取值范围在0-255之间），首先将其转换为标准的RGB值（sRGB），即除以255。
  2. 对 sRGB 值进行伽玛校正。即如果值小于或等于 0.03928，则将其除以 12.92；如果值大于 0.03928，则将其加上 0.055，然后除以 1.055，并且将得到的结果乘以自身（即乘方）。
  3. 计算相对亮度 L = 0.2126 * R’ + 0.7152 * G’ + 0.0722 * B’，这里的 R’、G’和B’ 是伽玛校正后的 sRGB 值。

如果你要单独计算相对亮度，建议使用任意一个 LLM 大模型来帮助你。

二、快速构建属于你的暗黑模式¶

颜色语义化¶

这是前提。颜色语义化可以有很多表现形式：

  • 建立颜色样式，应用在任意元素上
  • 建立 Design Token，应用在任意元素上

上面两点都需要和开发配合联动，但是 Design Token 的工程量更大。

一般来说，颜色语义化需要包括：

  • 通过颜色类型（背景、文本、图标、边框）和颜色角色（品牌、成功、危险、警告）来组织颜色
  • 引入了显著性（主要、次要、第三级）和交互模式（默认、悬停等状态）

在实际应用中，可能不需要用到这么复杂也能完成设计需要。比如在我负责的项目里，颜色样式设置成了下面这些：

  • 中性色：背景、上层背景、一级色、二级色、三级色、四级色、按钮底色等等
  • 彩色：主题色、错误色、链接色等等

❓ 这时候可能有人会说，不应该先建立一套完整的颜色体系吗？

是的，这个问题很好。但是现实不一定能给你足够的时间和空间来操作。作者所在的项目便是这样，缺乏一套颜色体系，但是好在有足够多的样式被应用到界面中。所以如果可以的话，还是推荐建立项目的颜色体系，更好的是使用 Design Token！

❓ 说了那么多，怎么快速建立暗黑模式呢？

不废话，简单来说就是构建好常用中性色和彩色即可！（甚至彩色也不用过度考虑）

中性色¶

中性色的调整其实不像想象中那么简单，在这里，需要满足这些目标：

  • 足够的识别度
  • 营造高端感

接下来将会告诉你怎么去达到这两个目标。首先，我们先将软件的颜色进行分类。一般来说中性色涉及：背景、文字、图标、边框。这个按需进行分类即可。将颜色分类好之后，我们下一步的行动才会清晰。

定调：营造高端感¶

背景色使用接近黑色单不是纯黑，甚至带有色调的黑色，这样可以加深视觉深度和质感。在暗色模式下，可以选择偏向深蓝或者深紫色。

  • 多个背景色的色系需要一致
  • 亮度提高，饱和度需要稍微降低
  • 避免使用阴影来区分层级高度，建议使用不同的明度来区分

文字：足够的识别度¶

根据 WCAG2.0 的规范要求，文字和背景的对比度至少要达到 AA 级别，也就是 4.5:1 的对比度。iOS 甚至是要求最低达到 7:1。在我负责的项目里，由于图标、边框和组件都是相对次要的，因此不被包含在考虑中。在实际开发中，根据项目对设计的精细度要求来规划具体哪些颜色需要符合标准即可。

基本上，我只需要调整好文字的颜色就能够把暗黑模式基本定稿了。那么，文字颜色具体是怎么调整出来的呢？这里又涉及到几个关键点：

  • 文字的亮度梯度如何界定？
  • 文字全部都要满足 WCAG2.0 标准吗？
  • 文字用纯色吗？

带着这些疑问，我对Material Design、Apple 和 Ant Design 这几个大的设计系统做了调研。

基于这个调研我们可以看到：

  • 文字一级和二级色必须要满足 WCAG2.0 的标准
  • 文字一级和二级的亮度梯度较大，二级和三级、三级和四级的变化较小

❓ Q：为什么文字仅一级和二级色满足 WCAG2.0 标准，三级和四级难道不重要吗？

A：没错，一级和二级通常用于界面中比较重要的文字，三级和四级色通常用于不重要的说明和禁用状态。因此，保证一级和二级达到 AA 标准是必须的。又因为需要和二级色拉开亮度差异，因此三级和四级色就不能保证达到可读性的标准了。

❓ Q：文字只能用纯色吗？

A：根据观察，苹果用了蓝色，有些平台则选择偏黄一些。在这一点上值得探究，当背景色作为冷色存在时，文字有多选择：

  • 和背景同系色
  • 和主题色同系色
  • 偏橙黄，减少在冷色背景下长时间阅读造成的眼睛疲劳

这里就看你是怎么考虑的了。

彩色和功能色¶

在调整彩色的时候，只需要遵循以下要求即可：

  • 主色调需要降低饱和度，适当提高亮度（这是因为高饱和的颜色在深色背景下会看起来饱和度更高，也就是霓虹灯效应。这样会降低可读性）
  • 用色节制，不要大面积的使用明亮的颜色

最后，简单回顾一下大致的流程：

  • 定调：确定背景色
  • 可读性：确定文字色
  • 微调：调整彩色

你看，暗黑模式也没有那么难吧。
